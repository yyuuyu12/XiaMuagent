一、整体结构调整
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

后台管理页面由原来的单页结构调整为 Tab 切换结构，拆分为两个独立 Tab：

  Tab 1：基础配置（原"AI 大脑配置"改名为"基础配置"）
  Tab 2：提示词配置（原提示词模板区块独立为单独 Tab）

两个 Tab 平级展示在后台顶部或左侧导航区域，点击切换。

提示词模块中，最高层级tab为全部、改写、行业热点。去掉提取分析
二级内容为行业的那些提示词，只在行业热点下面展示。分为两个内容，一是行业标题，调整前天的标签展示。二是提示词，提示词依然是支持在填空中修改，支持直接保存，支持多行输入。
搜索的提示词输入的框都支持换行等类似于代码格式的书写。

二、Tab 1：基础配置
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

基础配置 Tab 下包含三个独立配置模块，依次排列：

  1. 模型配置
  2. 授权码配置
  3. 会员配置

每个模块用卡片或分组标题区隔，视觉上清晰区分。

------------------------------------------------------------
2.1 模型配置
------------------------------------------------------------

功能说明：
  与现有"AI 大脑配置"上半部分完全一致，仅归属于新的 Tab 下。

字段列表：
  · AI 提供商（下拉选择）
      - 选项：OpenAI（或兼容接口）
      - 可扩展其他提供商

  · API Key（文本输入框，密码掩码显示）
      - 用于调用 AI 接口鉴权

  · 模型名称（文本输入框）
      - 示例默认值：gpt-5.4
      - 支持手动输入任意模型名称

  · Base URL（文本输入框）
      - 说明：兼容接口时填写，OpenAI 官方接口可留空
      - 示例：https://api.xxx.com/v1

  · Tikhub API Key（文本输入框）
      - 说明：用于抖音视频文案字幕提取服务
      - 注册地址提示：tikhub.io，有免费额度

操作按钮：
  · 【保存 AI 配置】按钮，点击后保存以上所有字段


------------------------------------------------------------
2.2 授权码配置
------------------------------------------------------------

功能说明：
  管理员可在后台生成、管理授权码。授权码供用户在前端个人中心激活，
  激活成功后用户可获得对应天数的免费使用资格以及每日免费次数上限。
  授权码具备两个核心维度：免费时长 和 免费次数。

【授权码列表区】

  展示内容（表格形式）：
    - 授权码（可复制）
    - 有效期配置（天数 或 已关闭）
    - 每日次数上限
    - 激活状态（未激活 / 已激活 / 已失效）
    - 激活用户（激活后显示对应用户信息）
    - 激活时间
    - 操作列：编辑 / 删除

  支持操作：
    - 批量生成授权码（输入数量，一次生成多个）
    - 单个新增授权码
    - 编辑已有授权码的时长和次数配置
    - 删除授权码

【新增 / 编辑授权码表单】

  字段 1：免费时长（必填）
    - 快捷选项按钮：【1天】【7天】【30天】【关闭授权】
    - 支持手动输入自定义天数（正整数）
    - 选择"关闭授权"时，输入框置灰不可用，授权码即为无效状态
    - 说明文字：授权码激活后，从激活当天起计算有效天数

  字段 2：每日免费次数上限（必填）
    - 默认值：30 次
    - 支持手动输入正整数
    - 输入范围建议：1 ~ 999
    - 说明文字：次数为每日可用上限，基于有效天数之上生效；
                若时长已关闭授权，次数配置无实际意义

  逻辑说明：
    - 授权码关闭（关闭授权）后，绑定该授权码的用户立即恢复为普通无授权状态
    - 修改次数配置时，可选择"仅对未激活授权码生效"或"对所有绑定用户实时生效"
      （具体策略按产品决策确认）
    - 授权码一旦激活不可重复激活（一码一用）

  操作按钮：
    - 【生成授权码】按钮（新增时）
    - 【保存修改】按钮（编辑时）

2.3 会员配置
------------------------------------------------------------

功能说明：
  管理员可在后台配置各会员套餐的价格以及支付页面的备注文案。
  配置保存后，前端会员开通页面实时同步展示最新价格与文案。

【套餐价格配置】

  套餐列表（固定四种，不可增删，仅价格可编辑）：

    套餐名称     有效期说明      价格输入框（单位：元）
    ─────────────────────────────────────────────
    日会员       有效期 1 天     [       ]
    周会员       有效期 7 天     [       ]
    月会员       有效期 30 天    [       ]
    永久会员     永久有效        [       ]

  字段规则：
    - 价格为正数，支持小数点后两位（如：9.90）
    - 不可为空，不可为 0

【支付页备注文案配置】

  字段：备注文案（文本输入框，支持多行）
    - 该文案将展示在前端会员开通页的支付按钮下方，红色加粗展示
    - 默认内容：请添加客服微信：Yu975196416
    - 管理员可随时修改

  操作按钮：
    - 【保存会员配置】按钮，点击后保存套餐价格及备注文案

三、Tab 2：提示词配置
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

功能说明：
  将现有后台"提示词模板"模块完整迁移至独立 Tab，功能不变，
  仅从原页面中剥离，作为独立的第二个 Tab 存在。

【分类筛选区】

  按钮组筛选：全部 / 改写 / 灵感 / 提取分析
  点击对应按钮，下方列表仅展示该分类的提示词

【提示词列表区】

  展示内容：
    - 提示词名称
    - 所属分类标签（改写 / 灵感 / 提取分析）
    - 是否为默认提示词（标记"默认"角标）
    - 操作列：编辑 / 删除 / 设为默认

【新增 / 编辑提示词表单】

  字段 1：名称（文本输入框）
    - 示例：灵感选题-默认

  字段 2：类型（下拉选择）
    - 选项：改写 / 灵感 / 提取分析

  字段 3：提示词内容（多行文本域）
    - 支持使用 {input} 占位符表示用户输入的内容
    - 示例说明文字：用 {input} 表示用户输入的内容

  操作按钮：
    - 【保存提示词】按钮
    - 【清空】按钮（清空当前表单内容）

4.1 个人中心页 —— 新增授权码激活模块
------------------------------------------------------------

改动位置：
  现有"个人中心"页面，账号状态卡片区域内

新增模块内容：

  标题：填写授权码

  输入框：
    - 占位提示文字：请输入授权码
    - 输入框右侧（或下方）有【立即激活】按钮

  激活逻辑：
    - 用户输入授权码后点击【立即激活】
    - 前端请求后端校验接口
    - 校验成功：
        · 提示"激活成功！已获得 X 天免费使用资格，每日可用 X 次"
        · 页面刷新或局部更新：今日剩余次数、会员等级状态同步更新
    - 校验失败，根据失败原因给出明确提示，场景如下：
        · 授权码不存在：提示"授权码无效，请检查后重新输入"
        · 授权码已被使用：提示"该授权码已被激活，无法重复使用"
        · 授权码已失效（关闭授权）：提示"该授权码已失效，请联系管理员"
        · 用户已有激活中的授权码：提示"您已有有效授权码，激活失败"（或允许覆盖，按产品决策）

  已激活状态展示：
    - 授权码已激活的用户，该模块改为展示当前授权状态：
        · 授权到期时间：XXXX年XX月XX日
        · 每日可用次数上限：XX 次
    - 不再展示输入框（或折叠隐藏，用小字提示"更换授权码"可展开）

  展示位置建议：
    - 放置于账号状态卡片内，"今日剩余次数"行上方或下方均可
    - 与"升级专业版"按钮区分层级，授权码为免费权益，会员为付费权益

4.2 会员开通页面（新增）
------------------------------------------------------------

入口：
  个人中心页"升级专业版"按钮点击后跳转

页面结构：

  【页面标题】
    - 标题文字：开通会员

  【套餐选择区】
    - 展示四个套餐卡片：日会员 / 周会员 / 月会员 / 永久会员
    - 每个卡片内容：
        · 套餐名称（大字显示）
        · 有效期说明（如：有效期 30 天）
        · 价格（如：¥29.90，从后台配置读取）
    - 默认选中：月会员（高亮边框或背景色区分）
    - 用户点击任意套餐卡片即切换选中状态

  【支付方式选择区】
    - 标题文字：选择支付方式
    - 两个支付选项（单选）：
        · 微信支付（图标 + 文字）
        · 支付宝支付（图标 + 文字）
    - 默认选中：微信支付
    - 当前阶段为 UI 占位展示，点击不触发真实支付流程
    - 后续接入微信/支付宝支付授权后在此对接

  【确认支付按钮】
    - 按钮文字：立即开通（¥XX.XX）—— 价格随选中套餐动态变化
    - 按钮样式：主色调，宽度占满屏幕底部或居中大按钮
    - 当前阶段点击后展示提示弹窗："支付功能即将上线，请添加客服微信：Yu975196416"

  【备注文案区】
    - 位置：确认支付按钮正下方
    - 样式：红色加粗字体，字号略小于按钮文字，左右留有适当边距
    - 内容：从后台"会员配置 - 支付页备注文案"读取，动态展示
    - 默认内容：请添加客服微信：Yu975196416

五、数据联动与业务逻辑说明
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 次数计算规则
   · 普通用户（无授权码、无会员）：每日享有系统默认免费次数
   · 授权码用户：在有效天数内，每日可用次数以"授权码配置次数"为准
   · 会员用户：会员有效期内次数不受限制（或按会员专属次数上限执行，按产品决策）
   · 若用户同时持有授权码与会员，优先以权益更高者为准（推荐取并集/最大值）

2. 今日剩余次数刷新规则
   · 每自然日 00:00 重置当日已用次数
   · 用户激活授权码或开通会员后，当日剩余次数立即更新，无需次日生效

3. 授权码修改策略
   · 修改时长/次数配置：默认仅对新生成的授权码生效
   · 若需对已激活授权码也实时生效，需在编辑弹窗中增加"应用到已激活用户"复选框
   · 关闭授权：立即对所有使用该授权码的用户生效，用户降级为普通状态

4. 会员价格修改
   · 后台修改价格后，前端会员开通页实时同步
   · 已开通中的会员不受价格调整影响，按购买时价格及有效期执行

5. 接口安全
   · 授权码激活接口需做防刷限制（同一用户/同一 IP 短时间内激活失败次数过多需锁定）
   · 授权码一码一用，激活成功后标记为已使用，不可再次激活
