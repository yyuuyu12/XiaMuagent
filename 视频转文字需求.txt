# 抖音链接转文字功能 — Claude Code 开发说明文档

## 项目背景

在现有小程序中新增「抖音视频转文字」功能。用户粘贴抖音分享链接后，系统自动完成：
视频下载 → 音频提取 → 语音转文字 → 返回文本结果。

---

## 整体架构概览

```
用户粘贴抖音链接（小程序前端）
        ↓
  阿里云服务器接收请求
        ↓
  调用 TikHub API 获取真实 MP4 地址并下载视频
        ↓
  调用本地 Windows 电脑的 ASR 服务（HTTP 长连接/轮询）
        ↓
  本地服务完成：音频提取 + 语音转文字
        ↓
  返回文字结果给阿里云服务器
        ↓
  阿里云服务器返回给小程序前端展示
```

## 模块一：阿里云服务器端（Node.js 推荐）

### 1.1 新增接口：`POST /api/video/douyin-to-text`

**请求体：**
```json
{
  "url": "麻辣烫85年的牛的抖音 - 抖音"
}
```

**响应体：**
```json
{
  "code": 0,
  "text": "这里是视频中转录出来的文字内容...",
  "duration": 120
}
```

### 1.2 接口逻辑

```
1. 接收前端传来的抖音链接
2. 调用 TikHub API 解析出真实 MP4 下载地址（见模块二）
3. 将 MP4 地址 + 任务 ID 转发到本地 Mac ASR 服务（见模块三）
4. 轮询或等待本地服务回调，获取转录文字
5. 将文字结果返回给前端
```

### 1.3 本地服务调用方式

本地 Mac 需要暴露公网（推荐使用 **frp 内网穿透** 或 **ngrok**），阿里云通过 HTTP 请求本地服务。

```javascript
// 阿里云 → 本地 Mac 调用示例
const response = await axios.post('http://<本地公网地址>/asr/transcribe', {
  taskId: uuid(),
  mp4Url: 'https://xxx.mp4'  // TikHub 返回的真实视频地址
}, {
  timeout: 300000  // 视频较长，超时时间设 5 分钟
});
```

## 模块三：本地 Windows 端 ASR 服务

### 3.1 环境要求（需在本地 Windows 安装）

| 工具 | 安装方式 | 用途 |
|------|---------|------|
| Python 3.9+ | 官网下载 https://www.python.org/downloads/ ，安装时勾选「Add to PATH」 | 运行 ASR 服务 |
| ffmpeg | 见下方详细说明 | 从 MP4 提取音频 |
| Whisper (OpenAI) | `pip install openai-whisper` | 本地语音转文字 |
| FastAPI + uvicorn | `pip install fastapi uvicorn` | 提供 HTTP 接口 |
| frp 或 ngrok | 见下文 | 内网穿透，暴露公网 |

#### ffmpeg Windows 安装步骤

1. 前往 https://www.gyan.dev/ffmpeg/builds/ 下载 `ffmpeg-release-essentials.zip`
2. 解压到 `C:\ffmpeg\`
3. 将 `C:\ffmpeg\bin` 添加到系统环境变量 PATH：
   - 右键「此电脑」→ 属性 → 高级系统设置 → 环境变量 → 系统变量 → Path → 编辑 → 新增 `C:\ffmpeg\bin`
4. 打开新的命令行窗口，运行 `ffmpeg -version` 验证安装成功

### 3.2 本地 ASR 服务代码（Python FastAPI）

请 Claude Code 创建文件：`local_asr_server/main.py`

```python
from fastapi import FastAPI
import whisper
import ffmpeg
import tempfile
import os
import httpx
import asyncio

app = FastAPI()
model = whisper.load_model("medium")  # 可选: tiny / base / small / medium / large

@app.post("/asr/transcribe")
async def transcribe(payload: dict):
    task_id = payload.get("taskId")
    mp4_url = payload.get("mp4Url")

    # 1. 下载 MP4 到临时目录
    with tempfile.TemporaryDirectory() as tmpdir:
        mp4_path = os.path.join(tmpdir, "video.mp4")
        audio_path = os.path.join(tmpdir, "audio.wav")

        # 下载视频
        async with httpx.AsyncClient(timeout=120) as client:
            response = await client.get(mp4_url)
            with open(mp4_path, 'wb') as f:
                f.write(response.content)

        # 2. 用 ffmpeg 提取音频（16kHz 单声道，Whisper 最优格式）
        (
            ffmpeg
            .input(mp4_path)
            .output(audio_path, ar=16000, ac=1)
            .overwrite_output()
            .run(quiet=True)
        )

        # 3. Whisper 语音识别
        result = model.transcribe(audio_path, language="zh")
        text = result["text"]

    return {"taskId": task_id, "text": text, "status": "done"}
```

### 3.3 启动本地服务

```bash
# 安装依赖（在命令提示符或 PowerShell 中运行）
pip install fastapi uvicorn openai-whisper ffmpeg-python httpx

# 启动服务（监听 8765 端口）
uvicorn main:app --host 0.0.0.0 --port 8765
```

> ```bash
> pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu
> ```
> 如果你的电脑有 NVIDIA 显卡，可以安装 CUDA 版本的 torch，转译速度会快很多，具体版本请查阅 https://pytorch.org/get-started/locally/

### 3.4 内网穿透配置（让阿里云能访问本地）

**方案 A：使用 frp（推荐，稳定）**

在阿里云服务器安装 frps，本地 Windows 安装 frpc：

1. 前往 Releases · fatedier/frp 下载 Windows 版 frpc（如 `frp_x.x.x_windows_amd64.zip`）
2. 解压后，创建 `frpc.ini` 配置文件：

```ini
# 本地 Windows：frpc.ini 配置示例
[common]
server_addr = <阿里云公网IP>
server_port = 7000

[asr_service]
type = http
local_port = 8765
custom_domains = asr.yourdomain.com
```

3. 运行：`frpc.exe -c frpc.ini`
4. 建议创建 Windows 开机自启任务（任务计划程序），确保电脑重启后自动恢复

**方案 B：使用 ngrok（快速测试）**

1. 前往 https://ngrok.com 下载 Windows 版
2. 运行：`ngrok http 8765`
3. 复制生成的公网地址，填入阿里云服务器的调用配置

### 4.2 前端请求代码（微信小程序）

```javascript
// pages/douyin-to-text/index.js
async function handleConvert() {
  const url = this.data.inputUrl.trim();
  if (!url) return wx.showToast({ title: '请粘贴抖音链接', icon: 'none' });

  this.setData({ loading: true, result: '' });

  try {
    const res = await wx.request({
      url: 'https://<你的阿里云域名>/api/video/douyin-to-text',
      method: 'POST',
      data: { url },
      timeout: 300000  // 5 分钟超时
    });

    if (res.data.code === 0) {
      this.setData({ result: res.data.text });
    } else {
      wx.showToast({ title: '转换失败，请重试', icon: 'none' });
    }
  } catch (e) {
    wx.showToast({ title: '网络超时，请重试', icon: 'none' });
  } finally {
    this.setData({ loading: false });
  }
}
```

## 开发顺序建议（给 Claude Code）

1. **先搭本地 ASR 服务**：创建 `local_asr_server/main.py`，本地测试能否正确转译一个本地 MP4 文件
2. **测试 TikHub API**：用 Postman 或脚本验证 TikHub 能正确返回 MP4 地址
3. **打通本地→阿里云链路**：配置 frp 内网穿透，确保阿里云能 HTTP 请求到本地服务
4. **完成阿里云服务端接口**：串联 TikHub 解析 + 调用本地 ASR + 返回结果
5. **完成小程序前端页面**：粘贴框 + 转换按钮 + 结果展示

## 注意事项

- **Whisper 模型大小**：`medium` 模型约 1.5GB，中文识别效果好；首次运行会自动下载
- **处理时长**：1 分钟视频大概需要本地处理 30-90 秒，前端需要有等待提示
- **Windows 保持开机不睡眠**：控制面板 → 电源选项 → 更改计划设置 → 「使计算机进入睡眠状态」改为「从不」
- **API Key 安全**：TikHub Key 只存在服务端，不要暴露在小程序前端代码中
- **视频版权**：仅限处理自己的或有权限的视频内容
