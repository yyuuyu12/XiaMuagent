## 1. 当前整体架构

```text
用户浏览器
  |
  | HTTPS
  v
Zeabur Node.js 后端
  | 认证、页面、API、数据库、任务状态
  |
  | HTTP，经 frp 公网穿透
  v
本地 Windows 机器
  |-- ASR 主服务      :8765  Whisper / edge-tts / 字幕 / 本地代理
  |-- IndexTTS 服务   :8766  声音克隆，异步任务模式
  |-- HeyGem V2 服务  :7861  数字人视频生成
  |-- frpc                 asr.yyagent.top / heygem.yyagent.top
```

公网穿透当前使用 frp，不是 ngrok：

| 域名 | 指向 | 用途 |
| --- | --- | --- |
| `http://asr.yyagent.top` | 本地 `127.0.0.1:8765` | ASR、TTS、字幕、IndexTTS 代理、数字人管理 |
| `http://heygem.yyagent.top` | 本地 `127.0.0.1:7861` | HeyGem 数字人视频生成 |

注意：当前 frp 域名使用 `http`。如果没有单独配置证书，不要在后台填 `https://asr.yyagent.top` 或 `https://heygem.yyagent.top`。

| --- | --- | --- |

### main.py 提供的关键接口

| 接口 | 说明 |
| --- | --- |
| `GET /health` | ASR 服务健康检查 |
| `POST /asr/transcribe` | 下载视频并 Whisper 转写 |
| `POST /tts/synthesize` | edge-tts 语音合成 |
| `POST /tts/indextts` | 旧同步 IndexTTS 代理，仍保留 |
| `POST /tts/indextts/submit` | 新异步 IndexTTS 提交任务 |
| `GET /tts/indextts/task/{task_id}` | 查询 IndexTTS 异步任务状态 |
| `POST /video/generate` | 代理 HeyGem 创建数字人视频任务 |
| `GET /video/task/{task_id}` | 查询数字人视频任务 |
| `GET /video/file/{task_id}` | 代理数字人视频文件，支持 Range |
| `POST /video/cancel/{task_id}` | 停止数字人等待/取消任务 |
| `POST /video/postprocess` | Whisper 时间轴 + ASS 字幕 + FFmpeg 烧录 |
| `/avatar/*` | 数字人素材上传、列表、删除、视频文件访问 |

### IndexTTS 异步模式

现在声音克隆不再让 Zeabur 一直等推理完成，避免 504：

```text
前端点击合成
  -> Zeabur /api/ai/tts
  -> ASR /tts/indextts/submit
  -> IndexTTS /tts/submit
  -> 立即返回 task_id
  -> 前端每 5 秒轮询 /api/ai/tts/indextts/task/{taskId}
  -> 完成后返回 audio base64
```

相关文件：

| 文件 | 关键点 |
| --- | --- |
| `backend/routes/ai.js` | `/api/ai/tts` 提交 IndexTTS 任务；`/api/ai/tts/indextts/task/:taskId` 轮询 |
| `backend/public/index.html` | `_pollIndexTTSTask(taskId)` 前端轮询 |
| `local_asr_server/main.py` | `/tts/indextts/submit` 和 `/tts/indextts/task/{task_id}` 代理到 8766 |
| `local_asr_server/indextts_server.py` | 维护 `_tasks`，后台推理，完成后保存 `audio` |

提速参数：

```python
max_text_tokens_per_segment=300
```

位置：`local_asr_server/indextts_server.py`

## 5. desktop_client：HeyGem 数字人服务

路径：`C:\AIClaudecode\desktop_client`

| 文件 | 作用 |
| --- | --- |
| `heygem_server_v2.py` | HeyGem V2 服务，端口 `7861` |
| `start_heygem.bat` | 手动启动 HeyGem |

关键接口：

| 接口 | 说明 |
| --- | --- |
| `GET /health` | HeyGem 服务健康检查 |
| `POST /video/generate` | 创建数字人视频任务 |
| `GET /video/task/{task_id}` | 查询任务状态 |
| `GET /video/file/{task_id}` | 返回生成视频文件 |
| `POST /video/cancel/{task_id}` | 取消任务 |

视频播放修复现状：

- HeyGem 生成完成后不再把大视频塞进 JSON base64。
- 服务端保存 `result_path`，通过 `/video/file/{task_id}` 文件接口提供视频。
- ASR 和 Zeabur 代理会转发 Range 请求。
- 前端下载 Blob 后创建本地 `blob:` URL，避免浏览器播放器显示 `0:00`。
- HeyGem 输出会做 `faststart`，把 MP4 `moov atom` 放到文件头，利于浏览器识别时长。

```text
C:\AIClaudecode\local_asr_server\frp\frp_0.61.0_windows_amd64\frp_0.61.0_windows_amd64\frpc.toml
```

当前配置：

```toml
serverAddr = "106.14.151.37"
serverPort = 7000

[[proxies]]
name = "asr"
type = "http"
localIP = "127.0.0.1"
localPort = 8765

[[proxies]]
name = "heygem"
type = "http"
localIP = "127.0.0.1"
localPort = 7861
customDomains = ["heygem.yyagent.top"]
```

健康检查命令：

```powershell
Invoke-WebRequest http://127.0.0.1:8765/health -UseBasicParsing
Invoke-WebRequest http://127.0.0.1:8766/health -UseBasicParsing
Invoke-WebRequest http://127.0.0.1:7861/health -UseBasicParsing
```

如果本地通、公网 404，通常是 `frpc` 没启动或代理未注册成功。

启动 frpc：

```powershell
cd C:\AIClaudecode\local_asr_server\frp\frp_0.61.0_windows_amd64\frp_0.61.0_windows_amd64
.\frpc.exe -c .\frpc.toml
```

看到下面两句就是代理注册成功：

```text
[asr] start proxy success

```

## 10. 主要业务流程

### 爆款克隆流程

```text
粘贴视频链接
  -> 提取原文案
  -> AI 改写
  -> 语音合成
  -> 数字人视频生成
  -> 后期字幕/封面
  -> 发布/保存
```

相关文件：

| 环节 | 主要文件 |
| --- | --- |
| 页面与状态 | `backend/public/index.html` |
| 提取/改写/语音/视频 API | `backend/routes/ai.js`、`backend/routes/extract.js`、`backend/routes/douyinToText.js` |
| 本地 ASR/TTS/字幕代理 | `local_asr_server/main.py` |
| IndexTTS | `local_asr_server/indextts_server.py` |
| HeyGem | `desktop_client/heygem_server_v2.py` |

### 声音克隆流程

```text
选择“我的声音”
  -> 前端带参考音频 base64
  -> /api/ai/tts 提交 IndexTTS 任务
  -> /tts/indextts/submit
  -> /tts/submit
  -> 前端轮询任务
  -> 返回 wav 音频
  -> 进入数字人步骤
```

### 数字人视频流程

```text
前端提交音频 + 数字人素材
  -> Zeabur /api/ai/video/generate
  -> HeyGem /video/generate
  -> 前端轮询 /api/ai/video/task/{taskId}
  -> 完成后拉取 /api/ai/video/stream/{taskId}
  -> 浏览器生成 Blob URL 播放
```

## 12. 常见问题排查

### 页面提示 IndexTTS 504

先看是否已经是最新异步模式：

```text
backend/routes/ai.js 中应有 /tts/indextts/submit
backend/public/index.html 中应有 _pollIndexTTSTask
local_asr_server/indextts_server.py 中应有 /tts/submit 和 /tts/task/{task_id}
```

再检查：

```powershell
Invoke-WebRequest http://127.0.0.1:8766/health -UseBasicParsing
Invoke-WebRequest http://asr.yyagent.top/health -UseBasicParsing
```

### 公网域名返回 frp 404

说明 frpc 没注册成功，重启 frpc：

```powershell
cd C:\AIClaudecode\local_asr_server\frp\frp_0.61.0_windows_amd64\frp_0.61.0_windows_amd64
.\frpc.exe -c .\frpc.toml
```

### 本地 8765 连不上

说明 ASR 没启动或已退出：

```powershell
netstat -ano | findstr "8765"
C:\AIClaudecode\local_asr_server\startup.ps1
```

### 数字人视频浏览器显示 0:00

重点检查：

```text
HeyGem /video/file/{task_id} 是否能返回 mp4
ASR /video/file/{task_id} 是否支持 Range
Zeabur /api/ai/video/stream/{taskId} 是否返回 video/mp4
前端是否使用 `${API_BASE}/ai/video/stream/${taskId}`
```

### 声音克隆太慢

当前已把：

```python
max_text_tokens_per_segment=300
```

如果还慢，优先缩短文案、缩短参考音频到 5-30 秒。后续可继续优化为更短文案分批生成或缓存参考音频特征。
