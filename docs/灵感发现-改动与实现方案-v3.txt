四、路径 A:链接解析与识别(技术实现)
------------------------------------------------------------

4.1 核心发现(基于实测)

抖音短链 https://v.douyin.com/xxx/ 的真实行为:

- HTTP GET 请求后返回 302 重定向
- Location 头里直接给出真实长链
- 长链格式:https://www.iesdouyin.com/share/user/{sec_user_id}?...
  或           https://www.iesdouyin.com/share/video/{aweme_id}?...
- Query 参数里还有 sec_uid=xxx(双保险)
- 展开过程不花钱、不需要 tikhub,自己用 axios 就能搞定

重要:tikhub 的 fetch_share_info_by_share_code 接口经实测
对格式非常挑,不接受纯短链和带短链的分享文本,只认带 @
和 : 符号的旧口令格式。因此本方案完全放弃该接口,改用
"自研短链展开 + 正则识别"方案。

4.2 链接解析完整代码(Node.js)

```javascript
const axios = require('axios');

const MOBILE_UA = 'Mozilla/5.0 (iPhone; CPU iPhone OS 15_0 like Mac OS X) ' +
                  'AppleWebKit/605.1.15 (KHTML, like Gecko) Version/15.0 ' +
                  'Mobile/15E148 Safari/604.1';

/**
 * 从用户粘贴的分享文本中抠出 URL
 * 支持短链和各种长链格式
 */
function extractUrl(text) {
  // 匹配优先级:长链 > 短链
  const patterns = [
    /https?:\/\/www\.iesdouyin\.com\/share\/(video|user)\/[A-Za-z0-9_\-]+/,
    /https?:\/\/www\.douyin\.com\/(video|user)\/[A-Za-z0-9_\-]+/,
    /https?:\/\/v\.douyin\.com\/[A-Za-z0-9]+/
  ];
  
  for (const pattern of patterns) {
    const match = text.match(pattern);
    if (match) return match[0];
  }
  return null;
}

/**
 * 展开短链,获取真实长链
 */
async function expandShortUrl(url) {
  if (!url.includes('v.douyin.com')) {
    return url;  // 已经是长链,直接返回
  }
  
  try {
    const res = await axios.get(url, {
      maxRedirects: 0,  // 不自动跟随,我们只要第一次 302 的 Location
      validateStatus: s => s === 301 || s === 302,
      headers: { 'User-Agent': MOBILE_UA },
      timeout: 10000
    });
    return res.headers.location;
  } catch (err) {
    throw new Error(`短链展开失败: ${err.message}`);
  }
}

/**
 * 从长链中提取 sec_user_id 或 aweme_id
 */
function classifyUrl(url) {
  // 先尝试从 path 提取
  const userInPath = url.match(
    /\/(?:share\/)?user\/([A-Za-z0-9_\-]+)/
  );
  if (userInPath) {
    return { type: 'user', sec_user_id: userInPath[1] };
  }
  
  const videoInPath = url.match(
    /\/(?:share\/)?video\/(\d+)/
  );
  if (videoInPath) {
    return { type: 'video', aweme_id: videoInPath[1] };
  }
  
  // 兜底:从 query 参数里找 sec_uid
  const secUidInQuery = url.match(/[?&]sec_uid=([A-Za-z0-9_\-]+)/);
  if (secUidInQuery) {
    return { type: 'user', sec_user_id: secUidInQuery[1] };
  }
  
  return { type: 'unknown' };
}

/**
 * 主入口:解析用户粘贴的任意内容
 */
async function resolveDouyinLink(userInput) {
  const url = extractUrl(userInput);
  if (!url) {
    throw new Error('未识别到有效的抖音链接,请复制分享链接后重试');
  }
  
  const expandedUrl = await expandShortUrl(url);
  const result = classifyUrl(expandedUrl);
  
  if (result.type === 'unknown') {
    throw new Error('链接格式不支持,请确保是抖音主页或视频');
  }
  
  return result;
}
```

4.3 能处理的输入全清单

resolveDouyinLink 函数能识别以下所有输入:

- 纯短链:https://v.douyin.com/xxx/
- 带分享文案的短链:1- 长按复制... https://v.douyin.com/xxx/ ...
- 视频长链:https://www.douyin.com/video/7xxx
- 主页长链:https://www.douyin.com/user/MS4w...
- 分享落地页:https://www.iesdouyin.com/share/user/MS4w...
- 分享视频页:https://www.iesdouyin.com/share/video/7xxx...

输出统一结构:
- 主页:{ type: 'user', sec_user_id: 'MS4w...' }
- 视频:{ type: 'video', aweme_id: '7xxxxx' }

五、路径 A:tikhub 接口对接(代码级)
------------------------------------------------------------

5.1 接口信息

Endpoint:
  GET https://api.tikhub.io/api/v1/douyin/app/v3/fetch_user_post_videos

鉴权:
  Header: Authorization: Bearer ${TIKHUB_TOKEN}
  Token 从环境变量读取,不要硬编码在代码里

参数:
  - sec_user_id  必填,从 resolveDouyinLink 返回值中取
  - max_cursor   整数,第一次传 0,翻页时传上次返回的 max_cursor
  - count        整数,建议 20(够用即可)
  - sort_type    整数,0=默认排序(按时间倒序)

注意:
  所有参数都必须传具体值,不能留空字符串。实测空字符串会
  返回 int_parsing 错误。

5.2 返回数据的关键字段结构

```

│   ├── aweme_id              视频 ID(缓存 key 用)

│   ├── item_title            精简标题(去掉 hashtag)
│   ├── duration              视频时长(毫秒)

│   │   │   ├── uri           视频唯一 URI
│   │   │   └── url_list[]    ★★ 无水印 mp4 地址列表(降级链)
```

5.3 重要发现(实测确认)

关键发现 1:无水印 mp4 地址直接在返回里
- video.play_addr.url_list 里有 4 条 URL
- 前两条是直连 CDN(douyinvod.com、zjcdn.com)
- 后两条是官方接口(amemv.com)作为兜底
- 这就是可以直接喂给本地 ASR 的 mp4 地址
- 因此路径 A 只需要 tikhub 调用 1 次,拿到列表后直接从
  每个视频的 play_addr.url_list 取 mp4 地址即可

关键发现 2:CDN 地址有时效性
- cdn_url_expired 字段,表示 CDN 地址过期时间(秒级时间戳)
- 实测过期时间约为请求后 1 小时
- 影响:列表不能缓存太久,用户勾选后必须立刻下载
- 对策:列表缓存最多 30 分钟(留出余量),超时就重拉

关键发现 3:URL 带签名,必须原样使用
- URL 里有 x-expires、x-signature、sign=xxx 等参数
- 这是签名过的临时地址
- 不能自己拼、不能删参数、不能改,原样传给下载器

关键发现 4:下载必须带移动端 UA
- 抖音 CDN 对 User-Agent 有检查
- 不带或用默认 UA 会返回 403
- 必须带:Mozilla/5.0 (iPhone; CPU iPhone OS 15_0 ...)

关键发现 5:视频时长差异巨大
- 实测返回里第一条 33 秒,第二条 111 秒,第四条 95 秒
- 本地 ASR 时间和视频长度成正比
- 对策:在前端按时长过滤或在后端设上限(见 5.5)

```javascript
/**
 * 把 tikhub 返回的单个视频对象,转成内部标准结构

function parseAwemeItem(aweme) {
  return {
    aweme_id: aweme.aweme_id,
    title: aweme.item_title || aweme.desc || '(无标题)',
    full_desc: aweme.desc,
    duration_ms: aweme.video?.duration || 0,
    duration_sec: Math.round((aweme.video?.duration || 0) / 1000),
    create_time: aweme.create_time,
    
    author: {
      nickname: aweme.author?.nickname || '',
      sec_uid: aweme.author?.sec_uid || '',
      signature: aweme.author?.signature || ''
    },
    
    cover_url: aweme.video?.cover?.url_list?.[0] || 
               aweme.video?.origin_cover?.url_list?.[0] || '',
    
    stats: {
      digg: aweme.statistics?.digg_count || 0,
      comment: aweme.statistics?.comment_count || 0,
      share: aweme.statistics?.share_count || 0,
      collect: aweme.statistics?.collect_count || 0
    },
    
    // ★ 下载用的 mp4 地址列表(降级顺序)
    play_urls: aweme.video?.play_addr?.url_list || [],
    
    // ★ 过期时间(毫秒戳)
    expire_at: (aweme.video?.cdn_url_expired || 0) * 1000
  };

/**
 * 拉取用户主页视频列表
 */
async function fetchProfileVideos(secUserId) {
  const res = await axios.get(
    'https://api.tikhub.io/api/v1/douyin/app/v3/fetch_user_post_videos',
    {
      params: {
        sec_user_id: secUserId,
        max_cursor: 0,
        count: 20,
        sort_type: 0
      },
      headers: {
        'Authorization': `Bearer ${process.env.TIKHUB_TOKEN}`
      },
      timeout: 15000
    }
  );
  
  if (res.data.code !== 200) {
    throw new Error(`tikhub 返回错误: ${res.data.message}`);
  }
  
  const awemeList = res.data.data?.aweme_list || [];
  const author = awemeList[0]?.author || {};
  
  return {
    author: {
      nickname: author.nickname || '',
      signature: author.signature || '',
      sec_uid: secUserId
    },
    videos: awemeList.map(parseAwemeItem)
  };
}
```

5.5 业务层过滤与排序

tikhub APP v3 接口的 sort_type 是否支持按热度排序不确定
(文档未明确说明)。建议做法:

- 先用 sort_type=0 拉 20 条
- 在后端按 digg_count 降序自排一遍
- 过滤掉时长超过 120 秒的视频(避免拖死 ASR)
- 取前 10 条返回给前端展示
- 前端默认勾选前 3 条(时长 < 90 秒的优先)

```javascript
function filterAndSort(videos) {
  return videos
    .filter(v => v.duration_sec > 0 && v.duration_sec <= 120)
    .sort((a, b) => b.stats.digg - a.stats.digg)
    .slice(0, 10);
}
```

6.1 下载单个视频(含 URL 降级)

url_list 里有 4 个地址,不是冗余,是降级链。依次尝试,
遇到错误(403、超时)就换下一个。

```javascript
const fs = require('fs');
const path = require('path');

const MOBILE_UA = 'Mozilla/5.0 (iPhone; CPU iPhone OS 15_0 like Mac OS X) ' +
                  'AppleWebKit/605.1.15 (KHTML, like Gecko) Version/15.0 ' +
                  'Mobile/15E148 Safari/604.1';

const DOWNLOAD_DIR = '/tmp/douyin_videos';

async function downloadVideo(playUrls, awemeId) {
  if (!playUrls || playUrls.length === 0) {
    throw new Error(`视频 ${awemeId} 没有可用的播放地址`);
  }
  
  fs.mkdirSync(DOWNLOAD_DIR, { recursive: true });
  const outputPath = path.join(DOWNLOAD_DIR, `${awemeId}.mp4`);
  
  for (const url of playUrls) {
    try {
      const response = await axios.get(url, {
        responseType: 'stream',
        headers: { 'User-Agent': MOBILE_UA },
        timeout: 30000,
        maxRedirects: 5
      });
      
      const writer = fs.createWriteStream(outputPath);
      response.data.pipe(writer);
      
      await new Promise((resolve, reject) => {
        writer.on('finish', resolve);
        writer.on('error', reject);
      });
      
      return outputPath;
    } catch (err) {
      console.warn(`下载失败,尝试下一个地址: ${err.message}`);
      continue;
    }
  }
  
  throw new Error(`所有播放地址都下载失败: ${awemeId}`);
}
```

```javascript
      await saveCachedASR(video.aweme_id, transcript);
    }
    
    results.push({
      ...video,
      transcript
    });
    
    // 清理 mp4 文件,不留垃圾
    fs.unlink(mp4Path, () => {});
  }
  
  return results;
}

// 缓存函数示例(用你实际用的缓存实现替换)
async function getCachedASR(awemeId) {
  return await redis.get(`asr:${awemeId}`);
}

async function saveCachedASR(awemeId, transcript) {
  // 永久缓存,不设过期
  await redis.set(`asr:${awemeId}`, transcript);
}
```

System Prompt:

十、打字机效果与轮询实现
------------------------------------------------------------

10.1 整体设计

- 前端轮询间隔:固定 2 秒
- 任务 done 或 failed 后:立即停止轮询
- 新文字到达后:旧文字淡出 300ms → 新文字逐字显示 50ms/字
- 不用 WebSocket/SSE,保持首期实现简单

10.2 前端实现(React 伪代码)

```javascript
function TaskCard({ taskId }) {
  const [task, setTask] = useState(null);
  const [displayedText, setDisplayedText] = useState('');
  const currentTextRef = useRef('');
  const typewriterRef = useRef(null);
  
  useEffect(() => {
    let isMounted = true;
    
    const poll = async () => {
      try {
        const res = await fetch(`/api/task/${taskId}`);
        const data = await res.json();
        if (!isMounted) return;
        
        setTask(data);
        
        // 检测思考过程文字变化
        if (data.thinking && data.thinking !== currentTextRef.current) {
          currentTextRef.current = data.thinking;
          runTypewriter(data.thinking);
        }
        
        // 任务结束,停止轮询
        if (data.status === 'done' || data.status === 'failed') {
          return;
        }
        
        // 继续轮询
        setTimeout(poll, 2000);
      } catch (err) {
        console.error('轮询失败', err);
        if (isMounted) setTimeout(poll, 5000);
      }
    };
    
    poll();
    
    return () => {
      isMounted = false;
      if (typewriterRef.current) clearInterval(typewriterRef.current);
    };
  }, [taskId]);
  
  const runTypewriter = (targetText) => {
    // 清除旧的打字机
    if (typewriterRef.current) clearInterval(typewriterRef.current);
    
    // 淡出旧文字(CSS 处理)
    setDisplayedText('');
    
    // 延迟 300ms 后开始打新字(等淡出动画)
    setTimeout(() => {
      let i = 0;
      typewriterRef.current = setInterval(() => {
        if (i >= targetText.length) {
          clearInterval(typewriterRef.current);
          return;
        }
        i++;
        setDisplayedText(targetText.slice(0, i));
      }, 50);
    }, 300);
  };
  
  return (
    <div className="task-card">
      <div className="task-header">{task?.title}</div>
      <div className="task-status">{task?.status}</div>
      {task?.status === 'running' && (
        <>
          <div className="thinking-text">{displayedText}</div>
          <div className="progress-bar">
            <div 
              className="progress-fill" 
              style={{ width: `${task.progress}%` }}
            />
          </div>
        </>
      )}
    </div>
  );
}
```

十四、工程注意事项
------------------------------------------------------------

1. 单视频管线零侵入
   新代码只能"调用"现有管线,不能"修改"它的入参出参。
   建议把现有管线封装成一个独立 service,新功能通过
   service.processSingleVideo(xxx) 这种方式调用。

2. ASR 是串行的,这是性能瓶颈
   不要试图并发,本地模型扛不住。
   3 条视频串行处理是总耗时的大头,必须靠缓存来摊薄成本。

3. 任务状态的实时性
   前端轮询间隔固定 2 秒,任务 done/failed 后立即停止。
   不用 WebSocket/SSE,首期保持简单。

4. 链接解析失败的兜底
   短链展开失败 → 提示用户"链接可能已失效,请重新复制"
   正则匹配不到 → 提示"未识别到有效的抖音链接"
   不要静默失败。

5. tikhub 接口失败的兜底
   - 401 认证失败 → 检查 TIKHUB_TOKEN 环境变量
   - 403/429 限流 → 等待重试,最多 3 次
   - 500 服务错误 → 标记任务失败,允许用户重试
   - 不要让整个路径 A 因此挂掉,保持单视频管线可用

6. CDN 地址过期的处理
   - 列表缓存时间(30 分钟)设置必须短于 CDN 过期时间(约 60 分钟)
   - 用户点击"开始拆解"时,检查 expire_at 是否已过期
   - 过期则重新拉 tikhub 接口(再走一次 fetchProfileVideos)
   - 用户全程操作应在缓存有效期内完成

7. 视频下载的 URL 降级
   play_urls 里有 4 个地址,必须依次尝试,不能只用第一个。
   403 错误通常是 UA 检查,确认 User-Agent 带了移动端 UA。

8. ASR 失败的兜底
   3 条视频里只要有 2 条转写成功,就允许进入后续 AI 分析。
   不要因为 1 条失败导致整个任务失败。
   给失败的视频一个占位 transcript:"(这条视频转写失败)"

9. GPT 返回 JSON 解析失败
   重试 1 次(temperature 调低一点),再失败就报错并允许
   用户重试任务。不要静默失败。
   建议用 try-catch 包裹 JSON.parse。

10. 思考过程文字不能写死在后端
    后端推送的应该是"阶段名 + 当前进度数字",前端根据阶段名
    从文案库里取对应的话术。这样以后改文案不用动后端代码。
    或者后端直接推送文案,但把文案常量集中在一个配置文件里。

11. 视频时长过滤
    duration > 120 秒的视频,ASR 时间会非常久。建议在
    filterAndSort 阶段就过滤掉,或在用户勾选时给出警告。

12. mp4 临时文件清理
    downloadVideo 下载的 mp4 文件在 ASR 完成后必须立即删除,
    否则 /tmp 会塞满。建议用 try-finally 确保清理。


十五、环境变量与密钥管理
------------------------------------------------------------

所有敏感信息必须通过环境变量读取,不要硬编码。

.env 文件示例(加入 .gitignore,不要提交):

  # tikhub API
  TIKHUB_TOKEN=你的_tikhub_api_key
  
  # OpenAI / GPT-4.5
  OPENAI_API_KEY=你的_openai_key
  OPENAI_MODEL=gpt-4.5-...
  
  # Redis(缓存)
  REDIS_URL=redis://localhost:6379
  
  # 本地 ASR 服务地址(如果是 HTTP 接口)
  ASR_SERVICE_URL=http://localhost:8080/asr
  
  # 临时文件目录
  DOWNLOAD_TMP_DIR=/tmp/douyin_videos

代码中读取:

```javascript
require('dotenv').config();

const TIKHUB_TOKEN = process.env.TIKHUB_TOKEN;
if (!TIKHUB_TOKEN) {
  throw new Error('TIKHUB_TOKEN 环境变量未设置');
}
```

安全准则:
- 永远不要把 key 写在代码里
- 永远不要把 key 提交到 git
- 永远不要把 key 发到聊天窗口、邮件、截图
- 测试和生产用不同的 key
- 如果 key 泄露过,立刻去 tikhub 后台重置
