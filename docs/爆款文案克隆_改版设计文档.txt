一、文档目标
----------------------------------------------------------------
本文档覆盖「爆款文案克隆」功能的两个核心页面：
  - Page A：链接输入页（用户粘贴视频链接并提交）
  - Page B：结果展示页（提取文案 + 操作区）

记录现状问题、改版方案、视觉规范与验收标准，供设计、
研发、产品三方对齐使用。


================================================================
二、现状问题分析
================================================================

【Page A  链接输入页】

  问题①  输入框与按钮视觉割裂
    当前状态：输入框和提取按钮之间没有明确的视觉关联，
    按钮颜色（粉橙）与整体橙色主色调不一致，偏淡偏粉，
    点击感不强。
    影响：用户操作路径不清晰，按钮吸引力弱。

  问题②  使用说明信息层级过重
    当前状态：使用说明卡片用了紫色背景块 + 紫色文字，
    视觉权重与输入区域相当甚至更高，喧宾夺主。
    影响：首先抓眼球的是说明文字而非操作入口。

  问题③  页面下半部分大面积空白
    当前状态：使用说明之下约 55% 的屏幕为空白。
    影响：页面头重脚轻，显得未完成。

  问题④  平台仅支持抖音，但无明显说明
    当前状态：标签写"视频链接（抖音）"，但输入框 placeholder
    同样只提到抖音，没有说明是否支持其他平台，也无快捷粘贴入口。
    影响：用户粘贴其他平台链接后才发现不支持，体验差。


【Page B  结果展示页】

  问题①  文案内容区无边界感，可读性差
    当前状态：大段文字直接铺满卡片，无段落分隔、无行高优化，
    文字与卡片边缘间距偏小。
    影响：长文本阅读压力大，用户难以快速扫读。

  问题②  「复制」按钮位置不合理
    当前状态：复制按钮独立放在卡片右上角，与正文距离远，
    且字体偏小（约 12sp），容易忽略。
    影响：核心操作（复制文案）不够突出，用户找不到。

  问题③  底部按钮组逻辑权重不清
    当前状态：「重新输入」和「一键原创改写」并排，但前者
    是破坏性操作（清空），后者是核心增值操作，两者高度一致，
    视觉权重相同。
    影响：用户容易误触「重新输入」，且核心 CTA 不够突出。

  问题④  页面下半部仍有大量空白
    同 Page A，结果卡和按钮之下约 50% 屏幕空白。
    影响：页面信息密度低，缺少引导用户继续操作的内容。

三、改版方案详述
================================================================

────────────────────────────────────────────
3.1  Page A  链接输入页  改版方案
────────────────────────────────────────────

3.1.1  页面整体结构调整

  改版后页面从上到下分为 4 个区：
  ┌─────────────────────────────┐
  │  顶部导航栏（返回 + 标题）    │  高度 56dp
  ├─────────────────────────────┤
  │  输入操作区                  │  主要区域
  │    - 平台 Tab 选择            │
  │    - 输入框                  │
  │    - 快捷粘贴按钮             │
  │    - 提取按钮                │
  ├─────────────────────────────┤
  │  折叠式使用说明               │  收起状态默认
  ├─────────────────────────────┤
  │  历史记录快捷入口（新增）      │  底部填充
  └─────────────────────────────┘

3.1.2  输入操作区

  新增平台 Tab：
    在输入框上方增加平台选择 Tab，当前支持「抖音」，
    预留「快手」「小红书」位置（置灰 + 即将上线标签）。
    Tab 选中态：主色橙 #F97316，下划线 2dp；
    未选中：#AAAAAA。

  输入框规范：
    高度：108dp（3行）
    背景：#FFFFFF
    圆角：12dp
    边框：1dp  #E0DDDA  聚焦态改为 1.5dp  #F97316
    Placeholder 文字：粘贴抖音视频链接或分享文字...
    字号：14sp  Regular  #BBBBBB

  快捷粘贴按钮（新增）：
    位于输入框右下角内侧，尺寸 64×28dp，圆角 6dp
    背景 #FFF0E6，文字「粘贴」14sp  #F97316
    点击后读取剪贴板并填入输入框

  提取文案按钮：
    高度：52dp，圆角 26dp（全圆角胶囊）
    背景色：#F97316（与品牌主色统一，去掉原来的粉橙）
    文字：🚀 提取文案  16sp  Bold  #FFFFFF
    按下态：背景 #D9650F（加深 10%）
    禁用态（输入框为空时）：背景 #DDDDDD  文字 #FFFFFF

3.1.3  折叠式使用说明

  默认收起，显示标题行「📌 使用说明  ›」可展开
  展开后显示 5 步说明，样式改为白底卡片 + 左侧橙色竖线
  去掉紫色背景，降低视觉权重

  收起态规范：
    高度：44dp，背景透明，文字 14sp Medium #666666
    右侧箭头图标 16×16dp，收起时向右，展开时向下

  展开态规范：
    背景 #FFFFFF，圆角 10dp，padding 16dp
    左侧橙色竖线：宽 3dp，圆角 1.5dp，颜色 #F97316
    每步文字：14sp  Regular  #555555  行高 1.8

3.1.4  历史记录快捷入口（新增，填充空白）

  标题：最近提取  14sp  Medium  #1A1A1A
  右侧：查看全部  12sp  #F97316
  展示最近 3 条提取记录，每条：
    - 文案前 40 字 + 省略号
    - 右侧时间标签（如「昨天」「3小时前」）
    - 点击可直接进入对应结果页
  无历史时隐藏此模块


────────────────────────────────────────────
3.2  Page B  结果展示页  改版方案
────────────────────────────────────────────

3.2.1  页面整体结构调整

  ┌─────────────────────────────┐
  │  顶部导航栏                  │  56dp
  ├─────────────────────────────┤
  │  文案展示卡片                │
  │    - 卡片标题行              │
  │    - 文案正文（可滚动）       │
  │    - 底部操作行              │
  ├─────────────────────────────┤
  │  改写风格选择器（新增）        │
  ├─────────────────────────────┤
  │  底部主操作按钮              │
  └─────────────────────────────┘

3.2.2  文案展示卡片

  卡片规范：
    背景 #FFFFFF，圆角 12dp
    padding：上 16dp  下 16dp  左右 16dp
    最大高度：屏幕高度 × 45%，超出内部滚动

  标题行：
    左侧：📄 提取的文案  15sp  Medium  #1A1A1A
    右侧：字数统计，如「共 248 字」12sp  #AAAAAA

  正文规范：
    字号：15sp  Regular  #333333
    行高：1.8（约 27px）
    段落间距：12dp
    文字与卡片左右边缘：已由 padding 保证 16dp 距离

  底部操作行（卡片内，靠底部）：
    分割线：0.5dp  #EEEEEE
    左侧：字数标签（同上，移至此处）
    右侧两个图标按钮：
      [复制全文]  图标 20dp + 文字「复制」11sp  #666666
      [分享]      图标 20dp + 文字「分享」11sp  #666666
    按钮间距 16dp

3.2.3  改写风格选择器（新增）

  位于文案卡片和底部按钮之间
  标题：选择改写风格  13sp  Medium  #1A1A1A
  横向滚动 Tag 列表（单选），4-6 个标签：
    口播带货 / 情感共鸣 / 干货科普 / 悬念钩子 / 轻松幽默
  Tag 规范：
    默认：背景 #F5F4F1  文字 #666666  圆角 16dp  padding 8×14dp
    选中：背景 #FFF0E6  文字 #F97316  边框 1dp #F97316

3.2.4  底部按钮组

  调整为主次结构，不再平等并排：

  次要按钮「重新输入」：
    宽度：屏幕宽度 × 30%（约 112dp）
    高度：52dp，圆角 26dp
    背景 #F5F4F1  文字 #666666  14sp
    点击弹出 确认弹窗（防误触）：
      「确定要清空并重新输入吗？」
      取消 / 确认

  主要按钮「一键原创改写」：
    宽度：屏幕宽度 × 65%（约 242dp）
    高度：52dp，圆角 26dp
    背景 #F97316  文字 #FFFFFF  16sp  Bold
    左侧 ✨ 图标
    加载态：按钮内显示转圈 loading 动画，文字改为「改写中...」

3.2.5  新增：改写结果区（改写完成后出现）

  改写完成后，页面追加展示改写结果卡片，与原文案卡片
  形成上下对比结构：

  改写结果卡片：
    卡片头部用橙色底 #FFF0E6 + 「✨ 改写后」标签区分
    操作行同原文案卡片（复制 / 分享）
    底部新增「再次改写」文字按钮（12sp  #F97316）

四、视觉设计规范（两页通用）
================================================================

【色彩】
  主色（品牌橙）       #F97316   — CTA 按钮、选中态、重点标签
  主色深（按下态）     #D9650F   — 按钮 pressed 状态
  主色浅（背景/填充）  #FFF0E6   — Tag 选中背景、提示区背景
  页面背景             #F0EDE8   — 全局米白底
  卡片背景             #FFFFFF   — 所有白色卡片
  辅助背景             #F5F4F1   — 次要按钮、折叠说明背景
  主文本               #1A1A1A   — 标题
  正文文本             #333333   — 卡片内正文
  辅助文本             #666666   — 描述、次要操作
  提示/标签文本        #AAAAAA   — 字数、时间、placeholder
  分割线               #EEEEEE   — 卡片内分割线
  边框                 #E0DDDA   — 输入框默认边框

【字体】
  主标题（页面级）     20sp  Bold    #1A1A1A
  卡片标题             15sp  Medium  #1A1A1A
  正文                 15sp  Regular #333333
  按钮文字（主）       16sp  Bold    #FFFFFF
  按钮文字（次）       14sp  Medium  #666666
  标签 / 辅助说明      13sp  Regular #666666
  时间 / 字数          12sp  Regular #AAAAAA
  Placeholder          14sp  Regular #BBBBBB

【间距】
  页面水平内边距       16dp
  卡片内 padding       16dp
  组件垂直间距         12dp
  按钮高度             52dp
  按钮圆角（胶囊）     26dp
  卡片圆角             12dp
  输入框圆角           12dp

【动效】
  按钮按下             scale(0.97)  duration 80ms  ease-out
  卡片展开             max-height 动画  duration 240ms  ease-in-out
  Loading 转圈         线形 spinner  24dp  颜色 #FFFFFF  1s/圈
  改写结果卡出现       fade-in + slide-up 12dp  duration 300ms

五、交互流程说明
================================================================

Page A 完整流程：

  1. 进入页面
     → 输入框聚焦状态，键盘自动弹起（可配置）
     → 历史记录（若有）显示在底部

  2. 用户粘贴链接
     → 方式一：点击「粘贴」快捷按钮，读取剪贴板
     → 方式二：长按输入框 → 系统粘贴

  3. 点击「提取文案」
     → 按钮进入 loading 态（转圈 + 文字变「提取中...」）
     → 全屏 loading 蒙层（不可点击，防重复提交）
     → 成功：跳转 Page B
     → 失败：toast 提示「链接解析失败，请检查链接格式」，
             返回可编辑状态

Page B 完整流程：

  1. 进入页面
     → 文案卡片以 fade-in 方式出现
     → 风格选择器默认选中第一个「口播带货」

  2. 用户可选操作：
     A. 复制文案 → 复制图标点击 → toast「已复制到剪贴板」
     B. 分享 → 调起系统分享面板
     C. 切换风格 → 点击不同 Tag
     D. 点击「一键原创改写」→ 按钮 loading 态 → 改写结果卡出现
     E. 点击「重新输入」→ 确认弹窗 → 确认后返回 Page A 并清空

  3. 改写完成后
     → 改写结果卡追加在原文案卡下方
     → 页面平滑滚动至改写结果卡
     → 底部按钮变为：「重新输入」+ 「复制改写版本」

六、边界状态处理
================================================================

  空状态（输入框为空时点击提取）
    → 输入框边框变红 #E24B4A，下方出现 12sp 红色提示：
      「请先粘贴视频链接」

  链接格式错误
    → toast（底部弹出，3s 自动消失）：
      「链接格式不正确，请粘贴抖音分享链接」

  网络异常
    → 全屏错误页（替换 loading）：
      图标 + 「网络连接失败」+ 「重试」按钮

  提取内容为空（视频无语音）
    → Page B 文案卡显示空态：
      「该视频未检测到语音内容，可尝试其他视频」

  文案超长（>500字）
    → 卡片固定最大高度，内部滚动
    → 底部渐变遮罩提示（#FFFFFF 至 transparent，24dp）

  改写失败
    → 改写按钮恢复可用
    → toast「改写失败，请稍后重试」

七、验收标准
================================================================

  Page A：

  [ ] 平台 Tab 显示「抖音」选中态（橙色下划线），
      其他平台置灰
  [ ] 输入框空态时「提取文案」按钮为禁用态（灰色），
      有内容后激活（橙色）
  [ ] 「粘贴」快捷按钮正确读取剪贴板并填入
  [ ] 使用说明默认收起，点击可展开/收起
  [ ] 历史记录有数据时展示，无数据时模块隐藏
  [ ] 提取中按钮 loading 动画正常，不可重复点击
  [ ] 失败 toast 3 秒后自动消失

  Page B：

  [ ] 文案正文行高 1.8，与卡片边缘保持 16dp 间距
  [ ] 字数统计数字与实际正文字数一致
  [ ] 复制按钮点击后 toast 显示「已复制到剪贴板」
  [ ] 风格 Tag 单选，选中态视觉正确
  [ ] 「重新输入」点击后弹出确认弹窗，取消不跳转
  [ ] 改写 loading 动画正常，改写结果卡 fade-in 出现
  [ ] 改写完成后页面自动滚动至结果卡
  [ ] 文案超过卡片最大高度时可内部滚动

八、待定事项
================================================================

  1. 平台扩展节奏：快手、小红书支持的上线时间？
     → 需产品确认排期，决定 Tab 是否显示「即将上线」标签

  2. 剪贴板读取权限：
     → iOS/Android 均需处理用户拒绝剪贴板权限的情况，
       需研发确认降级方案（引导用户手动粘贴）

  3. 改写风格来源：
     → 风格 Tag 是否由后端配置（可动态增删）？
       还是前端硬编码？需产品 + 研发确认

  4. 历史记录存储：
     → 本地缓存（最多 20 条）还是云端同步？
       云端需评估接口成本

  5. 改写次数限制：
     → 单条文案是否限制改写次数？
       如限制，需在 UI 上提示剩余次数
