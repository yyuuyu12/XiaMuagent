二、逐条改动说明
================================================================

────────────────────────────────────────────
改动 1  用户名
────────────────────────────────────────────

  当前：「你好，用户5539 👋」
  修改为：「你好，创作者 👋」

  执行方式：
    - 用户未设置昵称时，固定显示「创作者」
    - 用户已设置昵称时，显示对应昵称
    - 字段取值逻辑：nickname ?? '创作者'
    - 「用户 + 数字 ID」这种格式在任何情况下都不应出现


────────────────────────────────────────────
改动 2  次数提示区域
────────────────────────────────────────────

  当前：「登录后每日可免费生成5次」静态 pill
         ↑ 这个用户已登录，这行文案本身就是错误状态

  分两种登录状态处理：

  未登录态：
    pill 文字改为「登录领取每日免费次数」
    颜色：白底橙字，圆角 12dp

  已登录态（当前用户是这个状态）：
    去掉文字 pill，替换为进度条组件：

    组件结构（从左到右一行）：
      [进度条]                           [剩余 X 次]
      宽度占满 Banner 内容宽度 - 右侧数字区域

    进度条规范：
      高度：6dp
      圆角：3dp
      背景轨道色：rgba(255,255,255,0.30)
      填充色：#FFFFFF（白色）
      填充比例：已用次数 / 每日上限
      例：用了 1 次 / 上限 5 次 → 填充 20%

    右侧数字：
      「剩余 4 次」  13sp  Bold  #FFFFFF
      数字「4」加大为 15sp

    超限状态（剩余 0 次）：
      进度条填充色改为 #FFD580（暖黄警示）
      右侧文字改为「今日已用完」13sp  #FFD580
      下方追加小文字：「升级专业版获取无限次数」
      11sp  rgba(255,255,255,0.80)  带下划线可点击


────────────────────────────────────────────
改动 3  功能入口区  ——  主次分级
────────────────────────────────────────────

  当前：爆款文案克隆 和 灵感发现 两张卡片等宽等高并列，
        视觉权重完全一样。

  背景说明：
    爆款文案克隆 = 核心功能，用户主要用途
    灵感发现     = 次要功能，非重点入口

  修改方案：

  爆款文案克隆（主功能卡）：
    高度：72dp（比现在高 8dp）
    背景：#FFFFFF
    左侧图标背景：#FFF0E6（浅橙，与主色呼应）
    标题：15sp  Bold  #1A1A1A
    副标题：13sp  Regular  #AAAAAA
    右侧：橙色 › 箭头  #F97316

  灵感发现（次要功能入口）：
    不使用独立大卡，改为小型文字入口行
    样式：一行，左侧小图标（16dp）+ 文字 + 右侧箭头
    高度：44dp
    背景：透明（无卡片背景）
    文字：13sp  Regular  #888888
    放在主功能卡下方，视觉明显弱于主功能卡

  改动目的：
    让用户扫视页面时第一眼落在「爆款文案克隆」上，
    而不是两个功能平等竞争注意力。


────────────────────────────────────────────
改动 4  最近创作模块  ——  空态重设计
────────────────────────────────────────────

  当前空态问题：
    大白卡 + 大插图 + 橙色「去创作第一条文案」按钮
    卡片高度约 200dp，视觉权重超过主功能入口卡片
    按钮文案「去创作第一条文案」语义模糊

  修改方案：

  有历史记录时（正常态）：
    展示最近 3 条记录，每条一行：
      左侧：文案前 36 字 + 省略号  13sp  #333333
      右侧：相对时间（昨天 / 3小时前）11sp  #AAAAAA
    行高：48dp，行间细线分隔 0.5dp  #EEEEEE

  无历史记录时（空态）：
    去掉大插图和大按钮
    改为单行轻量空态：
      整行高度：48dp
      文字：「还没有克隆记录，」13sp  #AAAAAA
      紧跟文字链接：「去克隆第一条」13sp  #F97316  带下划线
      点击直接跳转爆款文案克隆功能页

    注意：
      · 不使用独立白色大卡包裹
      · 不放插图
      · 不放橙色按钮
      · 空态就是轻引导，不抢主功能的风头

  模块标题行：
    左：「最近克隆」  14sp  Medium  #1A1A1A
    右：「查看全部 ›」  12sp  #F97316
    标题行下方统一跟 12dp 间距再接内容


────────────────────────────────────────────
改动 5  统计数据模块  ——  背景色和视觉降权
────────────────────────────────────────────

  当前：白底卡片，与功能卡视觉层级完全相同

  修改方案：
    背景色：从 #FFFFFF 改为 #F5F4F1（浅灰）
    圆角：保持 10dp
    数字字号：从 ~24sp 缩小到 20sp  Bold  #F97316
    标签字号：保持 11sp  #AAAAAA

    右下角添加来源标注：
      「数据来源：平台累计统计」
      10sp  Regular  #CCCCCC  右对齐

  改动目的：
    浅灰底色让这个模块明显「比白色功能卡低一个层级」，
    用户视觉上会优先看白色卡片（功能入口），
    而不会把统计数字误认为是可操作的功能区。


────────────────────────────────────────────
改动 6  页面底部空白处理
────────────────────────────────────────────

  当前：统计模块以下约 50% 屏幕空白

  原因分析：
    最近创作空态卡片本身就占 200dp 左右，把统计模块
    推到了屏幕中部，中部以下全部空白。

  解决方式：
    步骤一：空态卡缩小（见改动 4），高度压缩到 48dp
    步骤二：统计模块自然上移，视觉上页面更紧凑

    如果步骤一完成后仍有较多空白：
    追加一个「热门行业」快捷入口模块：
      标题：「热门行业」  14sp  Medium  #1A1A1A
      横向滚动 Tag 列表（可滑动）：
        美妆 / 母婴 / 数码 / 食品 / 服装 / 宠物 / 家居
      Tag 样式：背景 #F5F4F1  文字 #666666  13sp  圆角 16dp
               padding 6×14dp
      点击 Tag 直接跳转灵感发现页并带入该行业参数

    这样既填充空白，又把灵感发现功能以轻量方式保留了入口。

五、验收重点（研发对照检查）
================================================================

  [ ] Banner 内，已登录用户显示进度条，不显示「登录后...」pill
  [ ] 进度条填充比例 = 已用次数 / 每日上限（实时计算）
  [ ] 用户名显示「创作者」，不出现任何数字 ID
  [ ] 爆款文案克隆卡高 72dp，视觉明显大于次要功能行
  [ ] 灵感发现为轻量单行，无白色卡片背景
  [ ] 最近克隆空态：单行文字 + 文字链接，无插图无大按钮
  [ ] 空态文字链接文案：「去克隆第一条」（非「去创作」）
  [ ] 空态点击直接跳转爆款文案克隆页
  [ ] 统计模块背景色 #F5F4F1（肉眼可见区别于白色卡片）
  [ ] 统计模块右下角有「数据来源：平台累计统计」灰色小字
  [ ] 热门行业 Tag 横向可滑动，点击跳转灵感发现并带入行业参数
  [ ] 页面底部无大面积空白（最长屏设备上内容区不低于 70%）
